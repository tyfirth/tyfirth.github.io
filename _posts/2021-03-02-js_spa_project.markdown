This module, like each module so far, was very fun and very challenging. It was helpful to have learned Ruby before to notice many similarities between the two languages, and once I got used to the syntactical differences it started to become more and more fun. However, this challenge wasn't without its share of challenges. Some of which had me researching, debugging, and banging my head against my desk for weeks! 

There were parts of the project that flew by. I felt like I was able to get the front end working fairly easily. Even though I don't have much experience with CSS, I was able to find the answers I needed fairly quickly. My biggest challenges came in establishing the back-end / front-end connection. Once again I ran into the error of "unpermitted parameters" which sent me down a rabbit hole working on strong parameters and nested attributes. I guess there were some fundamental understanding issues I had left over from the previous modules. Running into this issue in this module helped me work those kinks out. The problem in my case wasn't strong parameters or nested attributes, but was simply the lacking of rendering the JSON in the appropriate controller. My parameters and model attributes were able to stay VERY simple, and all I needed to do was render the appropriate relation information. Simple as that!

My other major challenge in this module was to figure out how to refactor my code from a single index file into different js model files...without breaking anything. This actually proved to be a useful challenge as well since it gave me an opportunity to review my current code and identify redundancy and also make it more readable. 

This was one of the more difficult module projects but was one of the most educational as well. Javascript is a very powerful language, but isn't the most user friendly. But I'm sure I will be more and more confident as I continue to practice and get experience.
